039    /**
040     * <p>A {@link ConcurrentMap} builder, providing any combination of these
041     * features: {@linkplain SoftReference soft} or {@linkplain WeakReference
042     * weak} keys, soft or weak values, size-based evicition, timed expiration, and
043     * on-demand computation of values. Usage example: <pre>   {@code
044     *
045     *   ConcurrentMap<Key, Graph> graphs = new MapMaker()
046     *       .concurrencyLevel(4)
047     *       .softKeys()
048     *       .weakValues()
049     *       .maximumSize(10000)
050     *       .expireAfterWrite(10, TimeUnit.MINUTES)
051     *       .makeComputingMap(
052     *           new Function<Key, Graph>() {
053     *             public Graph apply(Key key) {
054     *               return createExpensiveGraph(key);
055     *             }
056     *           });}</pre>
057     *
058     * These features are all optional; {@code new MapMaker().makeMap()}
059     * returns a valid concurrent map that behaves exactly like a
060     * {@link ConcurrentHashMap}.
061     *
062     * The returned map is implemented as a hash table with similar performance
063     * characteristics to {@link ConcurrentHashMap}. It supports all optional
064     * operations of the {@code ConcurrentMap} interface. It does not permit
065     * null keys or values.
066     *
067     * <p><b>Note:</b> by default, the returned map uses equality comparisons
068     * (the {@link Object#equals(Object) equals} method) to determine equality
069     * for keys or values. However, if {@link #weakKeys()} or {@link
070     * #softKeys()} was specified, the map uses identity ({@code ==})
071     * comparisons instead for keys. Likewise, if {@link #weakValues()} or
072     * {@link #softValues()} was specified, the map uses identity comparisons
073     * for values.
074     *
075     * <p>The returned map has <i>weakly consistent iteration</i>: an iterator
076     * over one of the map's view collections may reflect some, all or none of
077     * the changes made to the map after the iterator was created.
078     *
079     * <p>An entry whose key or value is reclaimed by the garbage collector
080     * immediately disappears from the map. (If the default settings of strong
081     * keys and strong values are used, this will never happen.) The client can
082     * never observe a partially-reclaimed entry. Any {@link java.util.Map.Entry}
083     * instance retrieved from the map's {@linkplain Map#entrySet() entry set}
084     * is a snapshot of that entry's state at the time of retrieval; such entries
085     * do, however, support {@link java.util.Map.Entry#setValue}.
086     *
087     * <p>The maps produced by {@code MapMaker} are serializable, and the
088     * deserialized maps retain all the configuration properties of the original
089     * map. If the map uses soft or weak references, the entries will be
090     * reconstructed as they were, but there is no guarantee that the entries won't
091     * be immediately reclaimed.
092     *
093     * <p>{@code new MapMaker().weakKeys().makeMap()} can almost always be
094     * used as a drop-in replacement for {@link java.util.WeakHashMap}, adding
095     * concurrency, asynchronous cleanup, identity-based equality for keys, and
096     * great flexibility.
097     *
098     * @author Bob Lee
099     * @author Kevin Bourrillion
100     * @since 2 (imported from Google Collections Library)
101     */

191       * or very often.
192       *
193       * @param size the maximum size of the map

195       * @throws IllegalArgumentException if {@code size} is not greater than zero
196       * @throws IllegalStateException if a maximum size was already set
197       * @since 8

202      public MapMaker maximumSize(int size) {

205        checkArgument(size > 0, "maximum size must be positive");
206        this.maximumSize = size;
207        this.useCustomMap = true;
208        return this;

211      /**
212       * Guides the allowed concurrency among update operations. Used as a
213       * hint for internal sizing. The table is internally partitioned to try
214       * to permit the indicated number of concurrent updates without
215       * contention.  Because placement in hash tables is essentially random,
216       * the actual concurrency will vary. Ideally, you should choose a value
217       * to accommodate as many threads as will ever concurrently modify the
218       * table. Using a significantly higher value than you need can waste
219       * space and time, and a significantly lower value can lead to thread
220       * contention. But overestimates and underestimates within an order of
221       * magnitude do not usually have much noticeable impact. A value of one
222       * is appropriate when it is known that only one thread will modify and
223       * all others will only read. Defaults to 16.
224       *
225       * @throws IllegalArgumentException if {@code concurrencyLevel} is
226       *     nonpositive
227       * @throws IllegalStateException if a concurrency level was already set
228       */

230      @Override

232        checkState(this.concurrencyLevel == UNSET_INT,
233            "concurrency level was already set to %s", this.concurrencyLevel);
234        checkArgument(concurrencyLevel > 0);
235        this.concurrencyLevel = concurrencyLevel;
236        return this;
237      }

285        checkState(keyStrength == null,
286            "Key strength was already set to %s", keyStrength);

372       * Specifies that each entry should be automatically removed from the
373       * map once a fixed duration has passed since the entry's creation.
374       * Note that changing the value of an entry will reset its expiration
375       * time.
376       *
377       * @param duration the length of time after an entry is created that it
378       *     should be automatically removed
379       * @param unit the unit that {@code duration} is expressed in
380       * @throws IllegalArgumentException if {@code duration} is not positive, or is
381       *     larger than one hundred years
382       * @throws IllegalStateException if the time to live or time to idle was
383       *     already set
384       * @since 8
385       */
386      @Beta
387      @Override
388      public MapMaker expireAfterWrite(long duration, TimeUnit unit) {
389        checkExpiration(duration, unit);
390        this.expireAfterWriteNanos = unit.toNanos(duration);
391        useCustomMap = true;
392        return this;
393      }

395      private void checkExpiration(long duration, TimeUnit unit) {
396        checkState(expireAfterWriteNanos == UNSET_INT,
397            "time to live of %s ns was already set", expireAfterWriteNanos);
398        checkState(expireAfterAccessNanos == UNSET_INT,
399            "time to idle of ns was already set", expireAfterAccessNanos);
400        checkArgument(duration > 0, "invalid duration: %s %s", duration, unit);
401      }

404        return (expireAfterWriteNanos == UNSET_INT)
405            ? DEFAULT_EXPIRATION_NANOS : expireAfterWriteNanos;
406      }

436      /**
437       * Specifies a listener instance, which all maps built using this {@code
438       * MapMaker} will notify each time an entry is evicted.
439       *
440       * <p>A map built by this map maker will invoke the supplied listener after it
441       * evicts an entry, whether it does so due to timed expiration, exceeding the
442       * maximum size, or discovering that the key or value has been reclaimed by
443       * the garbage collector. It will invoke the listener synchronously, during
444       * invocations of any of that map's public methods (even read-only methods).
445       * The listener will <i>not</i> be invoked on manual removal.
446       *
447       * <p><b>Important note:</b> Instead of returning <em>this</em> as a {@code
448       * MapMaker} instance, this method returns {@code GenericMapMaker<K, V>}.
449       * From this point on, either the original reference or the returned
450       * reference may be used to complete configuration and build the map, but only
451       * the "generic" one is type-safe. That is, it will properly prevent you from
452       * building maps whose key or value types are incompatible with the types
453       * accepted by the listener already provided; the {@code MapMaker} type cannot
454       * do this. For best results, simply use the standard method-chaining idiom,
455       * as illustrated in the documentation at top, configuring a {@code MapMaker}
456       * and building your {@link Map} all in a single statement.
457       *
458       * <p><b>Warning:</b> if you ignore the above advice, and use this {@code
459       * MapMaker} to build maps whose key or value types are incompatible with the
460       * listener, you will likely experience a {@link ClassCastException} at an
461       * undefined point in the future.
462       *
463       * @throws IllegalStateException if an eviction listener was already set
464       * @since 7
465       */
466      @Beta
467      @GwtIncompatible("To be supported")
468      public <K, V> GenericMapMaker<K, V> evictionListener(
469          MapEvictionListener<K, V> listener) {
470        checkState(this.evictionListener == null);
471    
472        // safely limiting the kinds of maps this can produce
473        @SuppressWarnings("unchecked")
474        GenericMapMaker<K, V> me = (GenericMapMaker<K, V>) this;
475        me.evictionListener = checkNotNull(listener);
476        useCustomMap = true;
477        return me;
478      }

482       * does not alter the state of this {@code MapMaker} instance, so it can be

485       * @return a serializable concurrent map having the requested features

495      /**
496       * Builds a caching function, which either returns an already-computed value
497       * for a given key or atomically computes it using the supplied function.
498       * If another thread is currently computing the value for this key, simply
499       * waits for that thread to finish and returns its computed value. Note that
500       * the function may be executed concurrently by multiple threads, but only for
501       * distinct keys.
502       *
503       * <p>The {@code Map} view of the {@code Cache}'s cache is only
504       * updated when function computation completes. In other words, an entry isn't
505       * visible until the value's computation completes. No methods on the {@code
506       * Map} will ever trigger computation.
507       *
508       * <p>{@link Cache#apply} in the returned function implementation may
509       * throw:
510       *
511       * <ul>
512       * <li>{@link NullPointerException} if the key is null or the
513       *     computing function returns null
514       * <li>{@link ComputationException} if an exception was thrown by the
515       *     computing function. If that exception is already of type {@link
516       *     ComputationException} it is propagated directly; otherwise it is
517       *     wrapped.
518       * </ul>
519       *
520       * <p>If {@link Map#put} is called on the underlying map before a computation
521       * completes, other threads waiting on the computation will wake up and return
522       * the stored value. When the computation completes, its new result will
523       * overwrite the value that was put in the map manually.
524       *
525       * <p>This method does not alter the state of this {@code MapMaker} instance,
526       * so it can be invoked again to create multiple independent maps.
527       *
528       * @param computingFunction the function used to compute new values
529       * @return a serializable cache having the requested features
530       */
531      // TODO(kevinb): figure out the Cache interface before making this public
532      <K, V> Cache<K, V> makeCache(
533          Function<? super K, ? extends V> computingFunction) {
534        return new ComputingConcurrentHashMap<K, V>(this, computingFunction);
535      }

537      /**
538       * Builds a map that supports atomic, on-demand computation of values. {@link
539       * Map#get} either returns an already-computed value for the given key,
540       * atomically computes it using the supplied function, or, if another thread
541       * is currently computing the value for this key, simply waits for that thread
542       * to finish and returns its computed value. Note that the function may be
543       * executed concurrently by multiple threads, but only for distinct keys.
544       *
545       * <p>If an entry's value has not finished computing yet, query methods
546       * besides {@code get} return immediately as if an entry doesn't exist. In
547       * other words, an entry isn't externally visible until the value's
548       * computation completes.
549       *
550       * <p>{@link Map#get} on the returned map will never return {@code null}. It
551       * may throw:
552       *
553       * <ul>
554       * <li>{@link NullPointerException} if the key is null or the computing
555       *     function returns null
556       * <li>{@link ComputationException} if an exception was thrown by the
557       *     computing function. If that exception is already of type {@link
558       *     ComputationException} it is propagated directly; otherwise it is
559       *     wrapped.
560       * </ul>
561       *
562       * <p><b>Note:</b> Callers of {@code get} <i>must</i> ensure that the key
563       * argument is of type {@code K}. The {@code get} method accepts {@code
564       * Object}, so the key type is not checked at compile time. Passing an object
565       * of a type other than {@code K} can result in that object being unsafely
566       * passed to the computing function as type {@code K}, and unsafely stored in
567       * the map.
568       *
569       * <p>If {@link Map#put} is called before a computation completes, other
570       * threads waiting on the computation will wake up and return the stored
571       * value. When the computation completes, its new result will overwrite the
572       * value that was put in the map manually.
573       *
574       * <p>This method does not alter the state of this {@code MapMaker} instance,
575       * so it can be invoked again to create multiple independent maps.
576       *
577       * @param computingFunction the function used to compute new values
578       * @return a serializable concurrent map having the requested features
579       */
580      @Override
581      public <K, V> ConcurrentMap<K, V> makeComputingMap(
582          Function<? super K, ? extends V> computingFunction) {
583        Cache<K, V> cache = makeCache(computingFunction);
584        return new ComputingMapAdapter<K, V>(cache);
585      }

587      /**
588       * A function which caches the result of each application (computation). This
589       * interface does not specify the caching semantics, but does expose a {@code
590       * ConcurrentMap} view of cached entries.
591       */
592      interface Cache<K, V> extends Function<K, V> {
593    
594        /**
595         * Returns a map view of the cached entries.
596         */
597        ConcurrentMap<K, V> asMap();
598      }
599    
600      /**
601       * Overrides get() to compute on demand.
602       */
603      static class ComputingMapAdapter<K, V> extends ForwardingConcurrentMap<K, V>
604          implements Serializable {
605        private static final long serialVersionUID = 0;
606    
607        final Cache<K, V> cache;
608    
609        ComputingMapAdapter(Cache<K, V> cache) {
610          this.cache = cache;
611        }
612    
613        @Override protected ConcurrentMap<K, V> delegate() {
614          return cache.asMap();
615        }
616    
617        @SuppressWarnings("unchecked") // unsafe, which is why this is deprecated
618        @Override public V get(Object key) {
619          return cache.apply((K) key);
620        }
621      }
